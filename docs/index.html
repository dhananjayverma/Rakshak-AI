🛡️ WebShield SDK

WebShield is an all-in-one, enterprise-grade application security shield and Web Application Firewall (WAF) middleware for Express.js applications. Instead of managing a disjointed collection of modules like helmet, cors, and express-rate-limit, WebShield binds all critical security barriers into a single, cohesive threat evaluation chain.

💡 Dynamic Defense: Rather than just blocking users, WebShield uses a Confidence Scoring Heuristic (0–100) to dynamically isolate attackers and issue custom challenges while keeping normal users unimpeded.

🚀 Installation

Install the package directly into your project via npm:

npm install webshield-sdk

⚡ Quick Start

Import the module and mount it as a global Express middleware. By default, it runs out of the box in strict protection mode:

const express = require('express');
const { webShield } = require('webshield-sdk');

const app = express();
app.use(express.json());

// Apply WebShield Protection
app.use(webShield({
  mode: "strict" // Blocks threat scores >= 75 automatically
}));

app.listen(3000, () => console.log('Secure server running on port 3000'));

🚦 Redis-Backed Rate Limiting

Configure distributed rate limits to prevent brute-force attacks across your server clusters. It features an automatic memory fallback:

app.use(webShield({
  redis: {
    enabled: true,
    host: '127.0.0.1',
    port: 6379,
    keyPrefix: 'webshield:'
  },
  rateLimit: {
    windowMs: 60000,
    max: 100,
    burstMax: 10 // Ban burst flood attempts
  }
}));

🛡️ Custom Web Application Firewall (WAF) & DSL Engine

Configure advanced, human-readable logical expressions to filter incoming requests dynamically:

app.use(webShield({
  waf: {
    enabled: true,
    rules: [
      // Logical DSL strings
      'if (path.includes("confidential")) block()',
      'if (headers["x-custom-block"] === "true") block()',
      'if (ip === "192.168.1.10") allow()'
    ]
  }
}));

💻 Command-Line Interface (CLI Tool)

WebShield comes bundled with a command-line interface helper. Use npx to trigger setup or visualizers:

# Initialize custom webshield.config.js configurations
npx webshield init

# Launch visual diagnostics server dashboard on port 9999
npx webshield dashboard

📡 Standalone Reverse Proxy Gateway

Filter external traffic before it touches your node application. Start the gateway script on port 8080 routing to backend application port 3000:

BACKEND_URL=http://localhost:3000 PROXY_PORT=8080 node bin/proxy.js

🛡️ Next-Gen Security Suite (Adaptive Protection)

Enable next-generation security configurations directly in your Zero-Config setup to trigger advanced behavioral heuristics, honeypots, and self-learning defenses:

app.use(webShield({
  stealthMode: true,          // Mask WAF blocks with delayed HTTP 200 responses
  consciousnessCheck: true,   // Parse cursor telemetry coordinates to detect click automation bots
  honeypot: {
    enabled: true             // Auto-injected hidden honeypots (e.g. /__trap)
  }
}));

🔌 Plugin Marketplace & Extensibility

WebShield features a plugin architecture allowing the community to build custom threat blocks. You can attach custom plugins during initialization:

const { webShield, plugins } = require('webshield-sdk');

app.use(webShield({
  plugins: [
    // Pre-loaded plugins
    plugins.botDetectionPlugin({ blockCrawlers: true }),
    plugins.ddosPlugin({ limit: 120 }),

    // Custom inline plugin
    async (req, res, config) => {
      if (req.headers['custom-threat-header']) {
        req.webShieldState.blocked = true;
        return res.status(403).send('Forbidden custom threat block');
      }
    }
  ]
}));
